# What I Can Do

I can help with software projects in this workspace by acting as a coding partner who can read, modify, and verify code directly.

## Code Help

- Explain how the codebase works.
- Trace bugs and identify likely root causes.
- Implement features and small improvements.
- Refactor code while preserving existing behavior.
- Add or update tests.
- Review code for bugs, risks, and regressions.

## Project Work

- Inspect the repository structure and give a quick tour.
- Update documentation and Markdown files.
- Run local commands, builds, linters, and tests when available.
- Help with Git tasks in a safe, non-destructive way.
- Work within the patterns already used by the project.

## Technical Guidance

- Answer programming questions.
- Compare implementation options and tradeoffs.
- Suggest debugging strategies.
- Help plan next steps for a feature or fix.

## OpenAI And Skills

- Help with OpenAI product and API questions using the available local skill workflow.
- Help create or install Codex skills when that matches the task.

## Working Style

- I can make changes directly in the repo instead of only describing them.
- I try to verify changes with local commands when practical.
- I avoid destructive actions unless you explicitly ask for them.
- I can keep explanations brief or go deeper, depending on what you want.
